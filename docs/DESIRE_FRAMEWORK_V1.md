# DESIRE_FRAMEWORK_V1

## 목적
주역 웹을 "좋은 말" 중심 서비스가 아니라,
**즉시 효용 → 성찰 전환 → 실행 1개**를 만드는 제품으로 운영한다.

## 핵심 전환 구조
1. 불확실성(불안) 인지
2. 통제 가능/불가 분리
3. If-Then 실행 1줄 생성
4. 저장 후 종료 (무한 소비 금지)

## 제품 원칙
- 점술 단정 금지
- 유한 세션(짧게 끝나야 함)
- 체류시간보다 성찰 완료율 우선
- 다크패턴(압박/가짜 긴급성/중독 루프) 금지

## KPI (운영)
- Activation: 괘상 뽑기 후 3분 성찰 체크 도달율
- Reflective Completion: 통제 가능 + If-Then 입력 완료율
- Unhealthy Loop Risk: 과다 반복 사용/압박감 피드백

## 현재 반영
- `/daily`에 3분 성찰 체크 도입
  - 잃기 싫은 것
  - 통제 가능/불가
  - If-Then 실행 1줄
  - 로컬 저장(`daily_reflections_v1`)
